Part 1: High-Level System Overview (For Context When Prompting Grok Build)
Core Philosophy

Mobile App = Lightweight data collector (fast, reliable capture on phone, minimal processing on device).
Desktop = Heavy workstation (full editing, collaboration, advanced tools, sharing/embedding).
Users can start on phone and continue/add data on desktop without losing work.
Everything ties into the existing Slate360 platform (unified_files, R2, entitlements, realtime, Stripe).

End-to-End Workflow (High Level)

Creation (Mobile or Desktop)
Mobile: User walks a space with phone → records video/photos with quality scoring → reviews → uploads.
Desktop: User can upload drone footage, 360 images, .PLY, or add to an existing space created on phone.

Processing
Upload goes through multipart resumable flow to R2.
Backend creates digital_twin_capture + assets → enqueues digital_twin_processing_jobs.
Trigger.dev runs twin.gaussian_splat (currently placeholder, will become real 3DGS + LiDAR fusion later).
Output: Published .spz model (or preview) stored in R2, linked via digital_twin_models.

Tracking & Delivery to User
Realtime updates on job status (queued → processing → ready).
User sees progress on mobile Processing Status screen or desktop Processing Queue / Space Detail.
When ready, published_model_id is set and model becomes viewable.

Deliverables User Gets
Interactive 3D model (Gaussian splat)
Pins, threaded comments, measurements
Branded shareable links (view / annotate / download)
Embed codes (for websites)
Punch List PDF export
Future: Sun study, BIM alignment overlays, etc.

Sharing & Leveraging
Create share tokens with roles, expiry, password, branding (logo for Pro).
Public viewer at /share/twin/[token] (atomic view counting).
Embed code for external websites (Pro feature).
Users (especially Pro) can create twins for clients and embed/brand them.

Editing & Iteration
Mobile: Basic pins/comments on light viewer.
Desktop: Full workstation — add more data (Upload More Data), re-process with different tier, align BIM, run sun studies, export PDFs, manage pins/comments in bulk.
Mixed workflow: Phone capture first → later add drone/360 on desktop → choose incremental update or new version (credits charged accordingly).


Mobile vs Desktop Split (Clear Difference)

Mobile: Capture-focused, fast, one-handed friendly, simple viewer. Goal = get data into the cloud quickly with good quality scoring.
Desktop: Full power — heavy editing, collaboration, advanced tools (Studio Toolkit), sharing/embedding, credit management, multi-asset workflows. This is where most value and monetization happens.

Credit System Integration

Credits are shown everywhere (header pill on mobile, prominent displays on desktop).
Every upload/processing action shows live cost preview.
Top-up available directly in app and dashboard (one-time Stripe purchases).
Low-credit warnings appear before expensive actions.
Pro tier gets more monthly credits + priority.
