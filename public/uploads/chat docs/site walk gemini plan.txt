1. The Multi-Layer Execution PipelineIngestion & Encryption Generation: The user opens the layout workspace and uploads a multi-page vector architectural plan set PDF. The file bypasses Next.js API routes and streams directly to Cloudflare R2 using a Presigned Multipart URL managed via @aws-sdk/lib-storage.  Job Enqueue Transaction: Upon upload verification, an API endpoint inserts a new row into plan_raster_jobs set to queued and immediately dispatches a background processing task over to Trigger.dev:  TypeScriptawait tasks.trigger("plan.rasterize", { planSetId, orgId, storageKey });
3.  **The Trigger.dev Rasterization Engine Task:** The worker spins up an isolated node instance. It fetches the source vector PDF from R2, initializes `pdfjs-dist` to parse internal sheet data, and executes an automated script loop to burst individual pages.
4.  **Coordinate Mapping & Tiling Execution:** The processing task checks if a parent vector position exists. If a user pins *Sheet B (Detail Blowout)* onto *Sheet A (Master Architectural Floor Plan)*, the worker maps the relative coordinate inputs and leverages `@napi-rs/canvas` + `sharp` to parse the vector lines into multi-resolution rasterized WebP image tiles. It writes these output layers back into R2 under an organized folder tree: `orgs/{orgId}/site_walk/rasterized/{sheetId}/{z}/{x}/{y}.webp`.
5.  **Realtime Stacking Re-Alignment UI:** Once the files are successfully written, the worker updates the data row in the database, setting `status = 'completed'` and appending the public `viewer_url` keys. Your custom client hook `usePlanSheetsRealtime.ts` intercepts this event change payload instantly, updating the Leaflet canvas viewport dynamically. The child plan sheet displays as a crisp, interactive vector rectangle overlay pinned precisely onto the master layout drawing.
